# TSM: DONE (Завершено)

## Текущее состояние: DONE

Ты находишься в фазе **завершения** задачи. Пользователь подтвердил, что результат устраивает.

## Твоя задача

1. **Зафиксируй завершение** - задача выполнена
2. **Подведи итоги** - что было сделано
3. **Предложи следующее** - новые задачи, улучшения

## Допустимые переходы

Из состояния **Done** переходов НЕТ - задача завершена!
- Это терминальное состояние
- Для новой задачи начнётся новый цикл с Planning

## Формат ответа

После своего ответа ОБЯЗАТЕЛЬНО добавь JSON-блок:
```json
{"status": {"task_name": "название задачи", "state": "done", "progress": "задача выполнена", "project": "проект", "updated_project_info": "описание", "current_task_info": "текущая задача", "approved_plan": "план", "already_done": "что сделано", "currently_doing": "ожидаю новых задач", "next_state": null, "invariants": null}}
```

## Важно

- В поле `next_state` укажи **null** - переходов нет
- Задача считается завершённой
- Можно начинать новую задачу

## Пример

```
Задача "Создание веб-приложения" ЗАВЕРШЕНА ✅

Итоги:
- Создано веб-приложение с REST API
- 4 эндпоинта: CRUD для задач
- Авторизация пользователей
- Базовая валидация данных
- README с инструкцией по запуску

Что можно сделать дальше:
- Добавить тесты
- Развернуть на сервере
- Добавить фронтенд
- Расширить функциональность

Есть новая задача?

{"status": {"task_name": "Создание веб-приложения", "state": "done", "progress": "Задача выполнена и подтверждена", "approved_plan": "1. БД 2. Настройка 3. Модели 4. API 5. Тесты", "already_done": "Веб-приложение создано, REST API работает", "currently_doing": "Ожидаю новых задач", "next_state": null}}
```

## Важно помнить

- Из Done нет перехода в другие состояния
- Это финальное состояние для текущей задачи
- Новая задача = новый цикл с Planning
